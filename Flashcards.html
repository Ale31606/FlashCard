1.- Alternativas al modelo OSI

Alternativas al modelo OSI

En el pasado, se utilizaron varios modelos de redes, como el intercambio secuenciado de paquetes o el intercambio de paquetes por Internet (SPX/IPX) y el sistema básico de entrada y salida de red (NetBIOS). Hoy en día, la alternativa principal al modelo de interconexión de sistemas abiertos (OSI) es el modelo TCP/IP.

2.- Capas del modelo OSI

Capas del modelo OSI

1.- Capa de aplicación

2.- Capa de enlace de datos

3.- Capa de física

4.- Capa de presentación

5.- Capa de red

6.- Capa de sesión

7.- Capa de transporte

2

3.- Capa de aplicacion

Capa de aplicacion

Esta es la única capa que interactúa directamente con los datos del usuario. Las aplicaciones de software, como los navegadores web y los clientes de correo electrónico, dependen de la capa de aplicación para iniciar las comunicaciones

3

4.- Capa de enlace de datos

Capa de enlace de datos

La capa de enlace de datos es muy similar a la capa de red, excepto que la capa de enlace de datos facilita la transferencia de datos entre dos dispositivos dentro la misma red. La capa de enlace de datos toma los paquetes de la capa de red y los divide en partes más pequeñas que se denominan tramas

4

5.- Capa física

Capa física

Esta capa incluye el equipo físico implicado en la transferencia de datos, tal como los cables y los conmutadores de red. Esta también es la capa donde los datos se convierten en una secuencia de bits, es decir, una cadena de unos y ceros.

5

6.- Capa de presentación

Capa de presentación

Esta capa es principalmente responsable de preparar los datos para que los pueda usar la capa de aplicación; en otras palabras, la capa 6 hace que los datos se preparen para su consumo por las aplicaciones. La capa de presentación es responsable de la traducción, el cifrado y la compresión de los datos.

6

7.- Capa de red

Capa de Red

La capa de red es responsable de facilitar la transferencia de datos entre dos redes diferentes. Si los dispositivos que se comunican se encuentran en la misma red, entonces la capa de red no es necesaria

7

8.- Capa de sesión

Capa de sesión

La capa de sesión es la responsable de la apertura y cierre de comunicaciones entre dos dispositivos. Ese tiempo que transcurre entre la apertura de la comunicación y el cierre de esta se conoce como sesión. La capa de sesión garantiza que la sesión permanezca abierta el tiempo suficiente como para transferir todos los datos que se están intercambiando; tras esto, cerrará sin demora la sesión para evitar el desaprovechamiento de recursos.

8

9.- Capa de Transporte

Capa de Transporte

Es la responsable de las comunicaciones de extremo a extremo entre dos dispositivos. Esto implica, antes de proceder a ejecutar el envío a la capa 3, tomar datos de la capa de sesión y fragmentarlos seguidamente en trozos más pequeños llamados segmentos. La capa de transporte del dispositivo receptor es la responsable luego de rearmar tales segmentos y construir con ellos datos que la capa de sesión pueda consumir.

9

10.- Cifrado

Cifrado

El cifrado es una forma de codificar datos para que solo las partes autorizadas puedan comprender la información.En términos técnicos, es el proceso de convertir texto sin formato legible por humanos en texto incomprensible, también conocido como texto cifrado.

10

11.- Comunicación en el modelo OSI

Comunicación modelo OSI

Las capas del modelo de interconexión de sistemas abiertos (OSI) están diseñadas para que una aplicación pueda comunicarse a través de una red con otra aplicación en un dispositivo diferente sin importar la complejidad de la aplicación y los sistemas subyacentes. Para ello, se utilizan varios estándares y protocolos para comunicarse con la capa superior o inferior.

11

12.- Conmutadores de red

Conmutadores de red

Un conmutador de red conecta dispositivos dentro de una red (a menudo a red de área local, o LAN*) y hacia adelante paquetes de datos hacia y desde esos dispositivos. A diferencia de un routerúnicamente, un conmutador envía datos al único dispositivo

12

13.- Correo electrónico

Conmutadores de red

Correo electrónico, comúnmente abreviado a “EMAIL,” es un método de comunicación que utiliza dispositivos electrónicos para entregar mensajes a través de redes informáticas. "Correo electrónico" se refiere tanto al sistema de entrega como a los mensajes individuales que se envían y reciben.

13

14.- El modelo de comunicación OSI

El modelo de comunicación OSI

El modelo de interconexión de sistemas abiertos (Open Systems Interconnection, OSI) es un marco conceptual que divide las funciones de comunicaciones de red en siete capas. El envío de datos a través de una red es complejo porque varias tecnologías de hardware y software deben funcionar de manera consistente a través de las fronteras geográficas y políticas.

14

15.- El modelo TCP/IP

El modelo TCP/IP

El modelo TCP/IP se compone de cinco capas diferentes (La capa física, La capa de enlace de datos, La capa de red ,La capa de transporte, La capa de aplicación).

15

16.- Enrutamiento

Enrutamiento

El enrutamiento de red es el proceso de seleccionar una ruta a través de una o más redes. Los principios de enrutamiento pueden aplicarse a cualquier tipo de red, desde redes telefónicas hasta transporte público.

16

17.- HTTP

HTTP

El Protocolo de Transferencia de Hipertexto (HTTP) es la base de la World Wide Web, y se utiliza para cargar páginas web utilizando enlaces de hipertexto. HTTP es un capa de aplicación protocolo diseñado para transferir información entre dispositivos en red y se ejecuta sobre otras capas de la red protocolo apilar.

17

18.- Importancia del modelo OSI

Importancia del modelo OSI

Las capas del modelo de interconexión de sistemas abiertos (OSI) encapsulan todos los tipos de comunicación de red en los componentes de software y hardware. El modelo se diseñó para permitir que dos sistemas independientes se comuniquen a través de interfaces o protocolos estandarizados basados en la capa de operación actual.

18

19.- Nota sobre los protocolos y modelos propietarios

Nota sobre los protocolos y modelos propietarios

Es importante tener en cuenta que no todos los sistemas y aplicaciones basados en Internet siguen el modelo TCP/IP o el modelo OSI. Del mismo modo, no todos los sistemas y aplicaciones basados en red sin conexión utilizan el modelo OSI ni ningún otro modelo.

20.- Paquetes

Paquetes

En networking, un paquete es un segmento pequeño de un mensaje más grande. Datos enviados a través de redes informáticas, como el Internet, se divide en paquetes. Estos paquetes son luego recombinados por la computadora o dispositivo que los recibe.

21.- Protocolo

Protocolo

En las redes, un protocolo es un conjunto de reglas para formatear y procesar datos. Los protocolos de red son como un lengua franca para los ordenadores.

22.- Protocolo de control de transmisión (TCP)

Protocolo de control de transmisión (TCP)

En las redes, un protocolo es un conjunto de reglas para formatear y procesar datos. Los protocolos de red son como un lengua franca para los ordenadores.

22

23.- SMTP

SMTP

El Simple Mail Transfer Protocol (SMTP) es un estándar técnico para transmitir correo electrónico a través de una red. Como otros protocolos de red, SMTP permite que las computadoras y los servidores intercambien datos independientemente de su hardware o software subyacente.

23

24.- User Datagram Protocol (UDP)

User Datagram Protocol (UDP)

El User Datagram Protocol, o UDP, es un protocolo de comunicación utilizado a través de Internet para transmisiones especialmente sensibles al tiempo, como reproducción de vídeo o DNS búsquedas. Acelera las comunicaciones al no establecer formalmente una conexión antes de transferir los datos.
